Learning Objective

Understand how computer memory manages variables. Learn the lifecycle, accessibility, and boundaries of variables defined in different scopes.

Concept Explanation

Variables exist in different memory segments based on where they are declared. Understanding these boundaries is critical for writing bug-free algorithms and optimizing system resources.

Common Misconception

"Local variables can be accessed anywhere after the function is executed."

Reality: False. When a function finishes execution, its Execution Context is popped off the Call Stack. All local variables declared inside are immediately destroyed.

Key Takeaways

  • Global Variables live until the program terminates. They use persistent memory.
  • Local Variables only exist within the specific function or block they are defined in.
  • Attempting to access a local variable outside its block results in a Scope/Compilation Error.

Think & Link

What happens if a local variable has the exact same name as a global variable?

This is called Variable Shadowing. The compiler/interpreter prioritizes the local scope. The local variable will "shadow" or hide the global variable within that function block.

Fun Fact

Every function call consumes a small amount of stack memory \( O(1) \). A recursive function without a base case will exhaust all memory, causing a fatal crash known as a Stack Overflow.
